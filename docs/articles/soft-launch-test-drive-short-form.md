# DevAudit Soft Launch: We Want Real Feedback

**Long-form parent:** Soft Launch: Take DevAudit for a Test Drive
**Platform:** LinkedIn, Dev.to
**Read time:** ~2 minutes

---

DevAudit is in soft launch, which means this is the right time to test it hard enough to improve it.

It started from a pretty practical problem: I was carrying QA across multiple projects at once and wanted AI agents to do as much of the work as possible while I still provided direction, guidance, oversight, and review.

That immediately raised the real question: how do you make that repeatable and consistent?

So the first step was the boring but necessary one: write the strategy and policy docs properly. Once those existed, it made sense to use AI to implement them wherever possible instead of leaving them as shelfware.

After a few iterations, DevAudit was what came out of that: a process to develop, test, and ship code consistently against different frameworks, with a release record that can actually be reviewed.

That is why DevAudit ended up as three connected parts instead of one:

- an SDLC framework
- the CLI and CI layer that applies it in a repo
- a portal that shows the release, evidence, and approvals in one place

We are inviting:

- developers and engineering leads
- founders and CTOs
- auditors and compliance reviewers
- contributors who want to open issues or PRs

What we want:

- real trial usage of the installer and portal
- concrete reports on confusing workflow steps
- feedback on missing or weak evidence
- suggestions for tighter control mapping
- pull requests for fixes that are already clear

The useful feedback is specific:

- “this article links to a doc that does not exist”
- “this onboarding claim says thirty seconds here and ten minutes there”
- “this release screen still does not prove the control you say it proves”
- “I can see what you’re aiming for, but this part still feels too heavy”

Soft launch is the stage where that kind of feedback still changes the product.

If you want to test an AI-assisted SDLC that tries to make compliance a byproduct of shipping, take it for a spin and tell us where it still falls short.
